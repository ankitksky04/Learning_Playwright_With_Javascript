# Learn Playwright Batch 2x

A structured JavaScript learning repository organized chapter-by-chapter, starting from basics and moving through variables, literals, operators, conditions, and user input.

## What This Repo Covers

This project is designed as a progressive practice path for beginners and early QA/automation learners. It includes:

- JavaScript fundamentals
- Identifier and naming rules
- `var`, `let`, `const`, scope, and hoisting
- Literals and data handling
- Operators and common logic questions
- `if/else` and `switch` decision making
- User input in Node.js (`readline` and sync prompt)

## Chapter Roadmap

1. `chapter_01_Basics`
Basic setup and JavaScript introduction files.

2. `chapter_02_Javascript_Concepts`
Core JavaScript concept starter files.

3. `chapter_03_Identifier_Literals`
Identifier rules and naming constraints.

4. `chapter_04_Javascript_Concepts`
Deep dive into variables, scope, functions, and hoisting.

5. `chapter_05_Literal`
Number, string, template literals, `null`, and `undefined`.

6. `chapter_06_Operator`
Arithmetic, comparison, strict/loose equality, logical, ternary, increment/decrement.

7. `chapter_07_If_else`
Conditional statements with practical examples and IQ-style exercises.

8. `chapter_08_Switch_Statement`
`switch` statement, `break`, `default`, grouped cases, and practice tasks.

9. `chapter_09_UserInput`
Node.js user input approaches:
- `68_User_Input.js`
- `69_Node_readline.js`
- `70_prompt_sync.js`

## Prerequisites

- Node.js (LTS recommended)
- VS Code (optional, but recommended)

Check versions:

```bash
node -v
npm -v
```

## How To Run Files

From the repository root, run any file with Node:

```bash
node chapter_01_Basics/01_Basics.js
node chapter_04_Javascript_Concepts/09_var_let_const.js
node chapter_09_UserInput/69_Node_readline.js
```

## Learning Approach

- Run one file at a time.
- Read output carefully.
- Modify values and conditions to observe behavior changes.
- Keep notes for confusing concepts (especially hoisting and loose vs strict comparison).

## Suggested Order

Follow files in numeric order (`01_`, `02_`, `03_` ...), chapter by chapter.

## Notes

This repository is practice-focused. Some files are intentionally small and repetitive to reinforce concepts through quick execution and experimentation.
